# 启动 Core

AutoLXB 的自动化能力由设备端 `lxb-core` 提供。App 负责启动、配置和展示状态；`lxb-core` 负责真正执行任务。

## Root 启动

Root 启动适用于已 Root 的设备，路径最短。

1. 打开 AutoLXB 首页。
2. 点击 **Root 启动**。
3. 在系统授权弹窗中允许 `su` 权限。
4. 等待首页显示 Core 已运行。

## ADB 启动

ADB 启动适用于未 Root 设备，需要先完成一次 Wireless ADB 配对。

1. 打开系统 **开发者选项**。
2. 开启 **USB 调试** 和 **无线调试**。
3. 回到 AutoLXB 首页，点击 **ADB 启动**。
4. 按引导进入无线调试配对页面。
5. 在通知栏提交配对码，等待配对和启动完成。

演示视频：<https://www.bilibili.com/video/BV1zXRWBiEgD>

## 如何确认启动成功

启动成功后，你应该能看到：

- 首页显示 Core 正在运行。
- 任务页可以刷新运行状态。
- 日志页能看到 Core trace。

## 常见问题

- **通知栏没有看到提交框**：到系统设置里检查 AutoLXB 的通知权限，确认已允许应用发送通知。
- **关闭前台应用后，服务中断，需要重新启动**：打开开发者选项，确认 **USB 调试** 仍然开启。
- **Root 启动失败**：检查 Root 管理器中是否拒绝或超时了 `su` 授权。
- **其他问题**：请到 GitHub Issues 反馈，并附上设备型号、系统版本、启动方式和日志截图。
