# 配置说明

配置页用于设置 AutoLXB 在本机上的运行方式。一般来说，第一次使用时只需要重点检查三类配置：**操控方式**、**模型配置**、**解锁与锁屏策略**。

## 操控方式配置

这部分决定 AutoLXB 如何在手机上点击、滑动和输入文字。

### 触控模式

常见选项包括：

- **Shell**：兼容性较好，适合大多数设备先尝试。
- **UIAutomator**：在部分设备上点击和滑动更精确。

如果你发现任务中点击无效、滑动失败或点错位置，可以在这里切换模式后重新测试。

### 输入模式

推荐安装并使用 **ADB Keyboard**。它通常比剪贴板或 shell 输入更稳定，尤其是中文、emoji 和较长文本输入场景。

如果没有安装 ADB Keyboard，AutoLXB 会尝试使用兜底输入方式，但部分 App 可能无法正确接收中文。

### 任务期间免打扰

任务执行时，新的通知、声音或弹窗可能干扰自动化。建议根据使用场景开启任务期间免打扰，减少中途被打断的概率。

## 设备端 LLM 配置

这部分决定 AutoLXB 使用哪个模型服务进行任务理解、页面观察和视觉执行。

需要填写：

- **API Base URL**：模型服务地址。
- **API Key**：模型服务密钥。
- **Model**：模型名称。

!!! warning "模型需要支持图像理解"
    AutoLXB 的视觉执行依赖截图输入。如果模型不支持图片，任务可能无法观察页面，也无法完成语义适配。

配置完成后，请点击测试，确认模型能正常处理图片并返回结果。

## 解锁与锁屏策略

这部分决定任务开始和结束时如何处理锁屏。

### 路线执行前自动解锁

开启后，如果任务开始时手机处于锁屏状态，AutoLXB 会尝试先解锁再执行任务。

当前解锁流程依赖正常的锁屏界面。如果锁屏上被音乐播放器、系统弹窗或其它覆盖层挡住，可能导致解锁失败。

### 任务结束后自动锁屏

开启后，如果任务执行前由 AutoLXB 解锁了手机，任务结束后会自动锁屏。

### 锁屏 PIN / 密码

当仅靠上滑无法解锁时，需要填写锁屏 PIN。目前更适合数字 PIN 场景。

## 路线相关配置

任务配置中会看到“按路线执行”开关。开启后，任务会优先尝试使用已经保存过的任务路线。

建议：

- 新任务第一次试跑时可以先关闭路线。
- 任务跑通并保存路线后，再开启路线执行。
- 如果路线经常失败，可以重新编辑路线或暂时关闭路线执行。

## App 地图相关选项

配置页中可能还保留地图来源或同步相关选项。当前用户日常使用更推荐围绕“单个任务的任务路线”来配置和保存路线；如果你只是创建定时任务或通知触发任务，通常不需要手动处理这些地图选项。
