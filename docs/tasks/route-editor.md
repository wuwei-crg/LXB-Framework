# 任务路线编辑

任务路线编辑页用于把一次任务执行轨迹整理成后续可复用的路线。你可以把它理解为“从刚才那次执行记录里，挑出以后还需要重复走的步骤”。

## 什么时候进入路线编辑？

建议在任务至少跑过一次后再进入路线编辑。即使任务最后失败，只要中间产生了有效操作记录，也可以打开路线编辑页查看并整理。

常见入口：

- 定时任务配置页中的路线编辑入口。
- 通知触发任务配置页中的路线编辑入口。
- 最近运行记录中进入对应任务的路线信息。

## 基本流程

1. 先运行一次任务。
2. 打开该任务的路线编辑页。
3. 查看最近一次捕获到的操作轨迹。
4. 把启动页等待、误触、无关弹窗处理等动作移出路线。
5. 保留真正代表任务主路径的步骤。
6. 点击 **手动保存路线**。
7. 再运行一次任务，确认路线能正常回放。

## 哪些动作应该删除？

建议删除：

- 误点、重复点击、无效等待。
- 与任务目标无关的弹窗处理。
- 只在某一次运行中偶然出现的页面。
- 进入错误页面后又返回的修正动作。

建议保留：

- 打开目标 App 后的固定入口。
- 进入固定 tab、固定页面的点击。
- 到达任务关键页面前的稳定跳转步骤。

## 回放后直接结束任务

“回放后直接结束任务”用于控制路线回放成功后的行为：

- 开启：路线回放成功后，任务直接结束。
- 关闭：路线回放成功后，继续进入后续视觉执行。

这个选项和路线一起保存。修改开关后，需要点击 **手动保存路线** 才会生效。

!!! warning "不要误开"
    如果你的路线只负责“进入目标页面”，后面还需要模型继续填写、判断或提交，就不要开启“回放后直接结束任务”。只有当路线本身已经完整完成任务时，才适合开启。

## 保存后的验证

保存路线后建议立刻再跑一次任务，观察：

- 是否更快到达目标页面。
- 是否跳过了不必要的模型判断。
- 是否出现路线回放失败。
- 失败后是否能正常降级到视觉执行。
