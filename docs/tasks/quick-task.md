# 快速任务

快速任务用于立即执行一次自然语言自动化需求。它不需要提前创建规则，也不需要配置时间，适合用来试跑任务和验证设备配置。

## 入口

在首页或任务页找到快速任务输入框，输入任务描述后提交即可。

## 适合场景

- 第一次测试 AutoLXB 是否能正常工作。
- 测试模型配置是否可用。
- 验证任务描述是否清晰。
- 临时执行一次不需要长期保存的操作。

## 示例

```text
打开微信，给文件传输助手发送一条消息：hello
打开 Bilibili，进入动态发布页面，发布一条测试动态
打开某 App，进入签到页面，完成签到
```

## 建议写法

尽量把任务写成“打开哪里 -> 找到什么 -> 做什么”的顺序：

```text
打开微信，进入文件传输助手，发送一条消息：hello
```

如果任务涉及特定联系人、群聊、商品或页面，也要写清楚：

```text
打开微信，进入名为“项目讨论”的群聊，回复刚刚发消息的人：收到
```

## 运行后看什么？

任务提交后，可以关注三件事：

1. 首页或任务页是否显示任务正在运行。
2. 手机界面是否按预期被打开和操作。
3. 日志页是否记录了执行过程和最终结果。

如果快速任务能稳定跑通，再考虑把同样的描述保存成定时任务或通知触发任务。

!!! warning "不要一次写太复杂"
    快速任务适合验证一条清晰路径。包含大量条件判断、多个 App 来回切换、或者需要长时间等待的任务，建议拆成更小的任务先测试。
