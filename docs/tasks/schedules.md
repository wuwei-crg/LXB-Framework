# 定时任务

定时任务用于让手机在指定时间自动执行任务。它适合固定时间、固定周期、路径比较稳定的自动化场景，例如每日签到、定时查看订单状态、定时发送提醒等。

## 创建定时任务

在任务页进入 **定时任务** 区域，选择新建或编辑任务。

填写时重点关注三类信息：

1. **任务要做什么**：任务名称、任务描述、可选操作文档。
2. **什么时候执行**：日期、时间、重复方式。
3. **怎么执行更稳定**：目标 App、是否按路线执行、是否录屏。

## 字段说明

| 字段 | 说明 | 是否必填 | 示例 |
| --- | --- | --- | --- |
| 名称 | 给这个定时任务起一个方便识别的名字。 | 否 | 点生椰拿铁 |
| 任务描述 | 告诉 AutoLXB 要完成什么任务。建议写清目标 App、目标页面和最终动作。 | 是 | 打开瑞幸咖啡，帮我点一杯生椰拿铁 |
| 用户操作文档 | 给模型看的补充步骤。适合写页面顺序、按钮名称、特殊要求。 | 否 | 进入瑞幸咖啡后，点击菜单，搜索生椰拿铁，选择热饮、大杯、堂食，最后付款 |
| 选择日期 | 单次执行时的日期。选择“每天”或“每周”时，主要以重复规则为准。 | 是 | 2026 年 4 月 10 日 |
| 选择时间 | 任务触发时间，使用 24 小时制。 | 是 | 11:00 |
| 重复 | 设置任务执行一次、每天执行，或每周指定几天执行。 | 是 | 每天 |
| 包名 | 指定任务要打开的 App。能选就尽量选，可以减少模型判断目标 App 的不确定性。 | 否 | 瑞幸咖啡 |
| 按路线执行 | 开启后，如果这个任务已经保存过路线，会优先回放路线。 | 是 | 开 |
| 录制任务屏幕 | 开启后会录制任务执行过程，方便失败后排查。 | 是 | 开 |

## 任务描述和操作文档怎么分工？

**任务描述** 写最终目标，尽量简短明确：

```text
打开瑞幸咖啡，帮我点一杯生椰拿铁
```

**用户操作文档** 写执行细节，适合补充模型可能不知道的页面路径：

```text
进入瑞幸咖啡后，点击菜单，搜索生椰拿铁。选择大杯、热饮、堂食。如果出现加购或推荐弹窗，关闭后继续下单。
```

## 使用建议

- 第一次创建定时任务前，建议先用快速任务跑一遍同样的任务描述。
- 如果任务会付款、下单或发送重要消息，先用低风险内容测试。
- 如果任务路径稳定，跑通后可以保存路线，再开启“按路线执行”。
- 修改时间或重复方式不会影响已经保存的路线；你可以放心调整执行时间。

## 导入任务

导入的定时任务默认是停用状态。导入后建议先检查任务描述、目标 App、执行时间和路线设置，确认无误后再启用。
