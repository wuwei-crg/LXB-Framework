# 任务导出与导入

任务导出与导入用于在不同设备之间迁移“任务配置 + 任务路线”。它不是单纯导出一串点击坐标，而是尽量保留任务说明、目标 App、路线步骤和必要的执行偏好。

## 什么时候需要导出？

适合导出：

- 你在一台设备上调好了一个定时任务，想放到另一台设备使用。
- 你想备份一个已经调好的通知触发任务。
- 你想把某个任务分享给别人，让对方导入后再本地适配。

不适合导出：

- 任务还没有跑通。
- 任务强依赖本机账号、时间、地理位置或临时状态。
- 任务里有付款、下单、发消息等高风险动作，但没有经过目标设备测试。

## 导出

导出入口在具体任务的配置页：

- 定时任务：从定时任务配置页导出。
- 通知触发任务：从通知触发任务配置页导出。

导出后，文件会保存到：

```text
Downloads/LXB/Tasks/lxb-task-portable-<时间戳>.json
```

导出文件会包含：

- 任务名称和任务描述。
- 目标 App 信息。
- 用户操作文档。
- 路线设置。
- 已保存的任务路线。
- “回放后直接结束任务”等路线行为配置。

## 不会导出的内容

为了避免把原设备的私人运行状态带到新设备，导出文件不会携带：

- 原任务的本机身份信息。
- 历史运行记录。
- 历史运行时间。
- 定时任务的具体执行时间。
- 通知规则的本机生效时段。

导入后，你需要在新设备上重新确认时间、启用状态和其它本机设置。

## 导入

导入入口在任务页的自动化区域旁。选择导出的 JSON 文件后，AutoLXB 会自动识别它是定时任务还是通知触发任务。

导入后：

- 会创建一个新的本地任务。
- 新任务默认停用，避免导入后立刻执行。
- 路线会绑定到这个新任务。
- 你需要检查任务描述、目标 App、触发条件和时间设置，再手动启用。

## 跨设备路线适配

有些路线步骤可以直接在新设备上复用，例如有明确文字或描述的按钮。也有一些步骤在原设备上只能靠坐标或图标位置完成，这类步骤导入后需要在新设备上重新适配。

首次运行导入任务时，AutoLXB 会尝试根据语义说明和当前界面重新找到目标控件，并把适配结果保存到新设备本地。

!!! warning "导入后一定要试跑"
    不同设备的屏幕尺寸、系统字体、App 版本和登录状态都可能不同。导入任务后，建议先手动触发一次低风险试跑，确认路线和任务行为都符合预期，再开启定时或通知触发。
